# Toollane – Idea Scorecard

Stand: Juni 2026
Zweck: Diese Datei bewertet distinctive Toollane-Ideen nach Potenzial, Risiko und strategischer Passung.

---

# 1. Ziel

Toollane soll langfristig nicht nur eine weitere Online-Tool-Sammlung sein.

Ziel ist:

```txt
mehr Wiedererkennung
mehr wiederkehrende Nutzer
stärkere Cluster
bessere Nutzerführung
höherer SEO-Wert
mehr Monetarisierungspotenzial
```

Diese Scorecard hilft dabei, aus vielen Ideen die besten nächsten Produktlinien auszuwählen.

---

# 2. Bewertungslogik

Jede Idee wird nach diesen Kriterien bewertet:

```txt
SEO Potential
Uniqueness
User Value
Repeat Usage
Monetization Potential
Implementation Effort
Technical Risk
Brand Fit
```

Bewertung:

```txt
10 = sehr stark
7–9 = gut bis sehr gut
4–6 = mittel
1–3 = schwach
```

Wichtig:

```txt
Diese Bewertung ist eine strategische Ersteinschätzung.
Später müssen echte Daten aus Search Console, Nutzerverhalten und Monetarisierung ergänzt werden.
```

# 7. Strategische Entscheidung

Aktuelle Empfehlung:

```txt
Nach AdSense zuerst Baby Name Companion Suite bauen.
```

Erste konkrete Umsetzung:

```txt
Baby Name Compare Tool
```

Warum dieses Tool zuerst:

```txt
es nutzt vorhandene Baby-Name-Daten
es stärkt den größten Cluster
es ist deutlich einzigartiger als ein Standard-Rechner
es kann intern stark mit Detailseiten und dem Generator verlinken
es ist relativ risikoarm
```

# 8. Offene Fragen für später

Vor Umsetzung klären:

```txt
Soll Baby Name Compare ein eigenes Tool werden oder Teil des Baby Name Generators?
Soll es in data/tools.ts als eigenes Tool erscheinen?
Soll es zusätzlich unter /baby-names verlinkt werden?
Wie viele Namen sollen verglichen werden können?
Sollen Ergebnisse lokal gespeichert werden?
Soll es sharebare Result Cards geben?
```

Aktuelle Tendenz:

```txt
Eigenes Tool als Teil der Name Lane.
Zusätzlich starke Verlinkung vom Baby Names Hub und Baby Name Generator.
```
